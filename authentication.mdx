# Authentication

The Zerion API uses **HTTP Basic Authentication**. Your API key serves as the username, and the password should be left empty.

## Getting Your API Key

1. Visit [developers.zerion.io](https://developers.zerion.io)
2. Sign up or log in to your account
3. Create a new API key from your dashboard

## Making Authenticated Requests

### Using the Authorization Header

Encode your API key in Base64 format as `api_key:` (note the colon with no password) and include it in the `Authorization` header:

```bash
curl --request GET \
  --url 'https://api.zerion.io/v1/wallets/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045/portfolio' \
  --header 'Authorization: Basic YOUR_BASE64_ENCODED_KEY'
```

### Encoding Your API Key

To encode your API key:

<CodeGroup>

```bash Bash
echo -n "your_api_key:" | base64
```

```javascript JavaScript
const encoded = btoa('your_api_key:');
// Use: `Basic ${encoded}`
```

```python Python
import base64
encoded = base64.b64encode('your_api_key:'.encode()).decode()
# Use: f'Basic {encoded}'
```

</CodeGroup>

## Example Requests

### cURL

```bash
# Using -u flag (automatically handles Basic auth encoding)
curl -u "your_api_key:" \
  "https://api.zerion.io/v1/wallets/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045/portfolio"
```

### JavaScript (fetch)

```javascript
const apiKey = 'your_api_key';
const encoded = btoa(`${apiKey}:`);

const response = await fetch(
  'https://api.zerion.io/v1/wallets/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045/portfolio',
  {
    headers: {
      'Authorization': `Basic ${encoded}`
    }
  }
);

const data = await response.json();
```

### Python (requests)

```python
import requests

api_key = 'your_api_key'

response = requests.get(
    'https://api.zerion.io/v1/wallets/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045/portfolio',
    auth=(api_key, '')
)

data = response.json()
```

## Error Responses

| Status Code | Description |
|-------------|-------------|
| `401` | Missing or invalid API key |
| `403` | API key doesn't have access to the requested resource |
| `429` | Rate limit exceeded |

### 401 Unauthorized Example

```json
{
  "error": "Unauthorized",
  "message": "Invalid or missing API key"
}
```

## Rate Limits

Rate limits and available features depend on your plan. Visit the [dashboard](https://dashboard.zerion.io) for details on your current limits and upgrade options.

If you exceed your rate limit, you'll receive a `429 Too Many Requests` response.

<Tip>
Check the `X-RateLimit-*` headers in API responses to monitor your usage.
</Tip>

## Security Best Practices

<Warning>
Never expose your API key in client-side code or public repositories.
</Warning>

- Store your API key in environment variables
- Use server-side code to make API requests
- Rotate your API key if it's ever compromised
- Use different API keys for development and production
